# 数学教材六步互动预习网站

把一本小学数学教材加工成一套可离线打开的学期预习网站。不要直接从 PDF 自由生成一大段 HTML，必须采用稳定的两层架构：

`PDF -> 教材规划 JSON -> 单元六步内容 JSON -> 校验 -> 固定模板建站 -> 测试 -> ZIP`

AI 负责理解教材、设计教学内容和建立来源映射；确定性的 Python、CSS 和 JavaScript 负责统一生成、测试和打包网页。

## 六步预习法

每个单元统一执行：

1. **看全貌**：5—8 张知识卡组成单元知识地图；
2. **测基础**：1 道前置知识题 + 1 道典型误解预测题；
3. **想明白**：3—5 步苏格拉底式引导，先思考、分级提示、最后显示教材结论；
4. **验理解**：“认一认、说一说、用一用”三关逐步解锁；
5. **练迁移**：根据真实错误生成错因诊断，并完成母题、换数字、换问法、换情境；
6. **定重点**：汇总已理解内容、仍需关注的问题、正式课堂重点、想问老师的问题和 1—3—7 复习计划。

打印知识卡、错因卡、一题三变和完整成果单属于第六步的成果保存功能，不是第六步的学习目标。

## 默认工作流

1. 检查 PDF 文字层、扫描质量、目录、页码和关键图形；
2. 创建 `content/book.json` 与 `content/units/unit-XX.json`；
3. 每单元完整生成“看、测、想、验、练、定”六步内容；
4. 运行内容校验，修复定义、答案、错因映射和页码问题；
5. 使用固定模板生成学期总览、单元页、打印页和报告；
6. 运行网站测试；
7. 打包为离线 ZIP，并诚实列出待人工核验项。

## 单元内容硬要求

- 5—8 张知识卡；
- 正好 2 道前置小测；
- 3—5 步苏格拉底式引导和 1 个自我解释总结；
- 认一认、说一说、用一用三关；
- 每个错误选项映射到具体错因；
- 母题、换数字、换问法、换情境共 4 个练习；
- 第六步动态汇总已理解、待关注和正式课堂重点；
- 每项内容都绑定教材页码；
- 不超纲、不押题、不把预习变成大量刷题。

## 失败处理

- PDF 缺页或关键图形模糊：列出缺失或不确定项，暂停对应单元；
- 目录冲突：只向用户做一次合并确认；
- 内容结构不完整：补齐六步数据后再建站，不交付缩水版；
- 代码测试失败：修复后重跑，不交付“可能可用”的网页；
- 内容过多：减少装饰动画，不删除六步中的任何一步；
- 离线网页不得假装具有实时 AI 自由问答或开放文本语义评分能力。

## 示例调用

```text
请使用 math-semester-preview-site 技能处理我上传的五年级下册数学教材 PDF。按“看—测—想—验—练—定”六步预习法，自动识别目录并生成学期总览和每单元独立网页。除非教材版本或页码不确定，否则不要反复提问。最终输出离线 ZIP、教材来源报告和内容核验报告。
```
